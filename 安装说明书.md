## 项目简介

**垃圾分类助手** 是一个基于 Web 的智能垃圾分类识别系统，支持：
- 📷 **图片识别**：上传垃圾图片，AI 自动识别分类
- 🔍 **文字搜索**：输入物品名称查询分类信息
- 📝 **历史记录**：查看个人识别历史
- ⚠️ **纠错反馈**：对识别结果进行反馈，管理员审核优化
- 📊 **高频易错**：展示高频易错物品，提升分类准确率

**技术栈**：
- 后端：Flask 3.x + SQLAlchemy
- AI 模型：YOLOv8 (Ultralytics)
- 前端：Bootstrap 5.3 + Jinja2
- 数据库：SQLite (可扩展 MySQL/PostgreSQL)

## 安装步骤

### 方式一：Conda 环境安装（推荐）

#### 步骤 1：克隆项目代码
```bash
git clone <项目仓库地址>

```

#### 步骤 2：创建 Conda 环境
```bash
conda env create -f environment.yml
```

#### 步骤 3：激活环境
```bash

```

### 方式二：pip 直接安装

#### 步骤 1：创建虚拟环境

## 配置说明

### 1. 环境变量配置

在项目根目录创建 `.env` 文件（可选，使用默认值可跳过）：

```bash
# .env 文件内容

# Flask 会话密钥（生产环境请修改为随机字符串）
SECRET_KEY=YourSecretKeyHere123456

# 数据库连接 URI（默认使用 SQLite）
DATABASE_URL=sqlite:///instance/app.db

# MySQL 示例（如需切换数据库）
# DATABASE_URL=mysql://username:password@localhost:3306/rubbish_db

# PostgreSQL 示例
# DATABASE_URL=postgresql://username:password@localhost:5432/rubbish_db
```

### 2. 目录结构确认

安装完成后，确保以下目录和文件存在：

```
项目根目录/
├── app/
│   ├── __init__.py          # ✅ 应用工厂
│   ├── models.py            # ✅ 数据模型
│   ├── routes.py            # ✅ 路由逻辑
│   ├── best.pt              # ✅ YOLO 模型文件 (约 22MB)
│   ├── static/
│   │   └── images/
│   │       └── avatars/     # 用户头像存储
│   └── templates/           # HTML 模板
├── instance/
│   ├── app.db               # (首次运行后自动生成)
│   └── uploads/             # (首次运行后自动生成)
├── run.py                   # ✅ 启动入口
├── environment.yml          # ✅ 环境配置
└── .env                     # (可选) 环境变量
```

### 3. 模型文件检查

确认 `app/best.pt` 文件存在且完整（约 22MB）：
```bash
ls -lh app/best.pt
```

如文件缺失，请联系项目维护者获取或重新训练 YOLO 模型。

## 常见问题

### Q1: 导入错误 `ModuleNotFoundError: No module named 'flask'`
**解决方案**：
```bash
# 确认已激活虚拟环境
conda activate rubbish  # 或 source venv/bin/activate

# 重新安装依赖
pip install flask flask-sqlalchemy python-dotenv ultralytics
```

---

### Q2: YOLO 模型加载失败
**错误信息**：`YOLO 模型未加载` 或 `best.pt not found`

**解决方案**：
1. 检查模型文件是否存在：
   ```bash
   ls -lh app/best.pt
   ```
2. 确认文件大小约为 22MB（文件损坏会导致加载失败）
3. 如文件缺失，从备份恢复或重新下载

---

### Q3: 数据库初始化失败
**错误信息**：`sqlite3.OperationalError: unable to open database file`

**解决方案**：
1. 确保 `instance` 目录有写权限：
   ```bash
   mkdir -p instance/uploads
   chmod -R 755 instance
   ```
2. 删除旧数据库文件重新生成：
   ```bash
   rm instance/app.db
   python run.py
   ```

---

### Q4: 图片上传失败
**错误信息**：`无法保存上传文件`

**解决方案**：
1. 检查上传目录是否存在：
   ```bash
   mkdir -p instance/uploads
   ```
2. 确认目录权限：
   ```bash
   chmod 755 instance/uploads
   ```
3. 检查上传文件格式（仅支持：jpg, jpeg, png, bmp）

---

### Q5: 端口被占用
**错误信息**：`Address already in use`

**解决方案**：
1. 查找占用端口的进程：
   ```bash
   # Linux/macOS
   lsof -i :5000
   
   # Windows
   netstat -ano | findstr :5000
   ```
2. 终止进程或修改端口：
   ```bash
   # 修改 run.py 中的端口号
   app.run(host='0.0.0.0', port=8080, debug=True)
   ```

---

### Q6: 首次运行页面空白或样式丢失
**解决方案**：
1. 清除浏览器缓存
2. 检查 Bootstrap CDN 是否可访问（需联网）
3. 确认 `templates/base.html` 中 CSS/JS 引用路径正确

## 管理员初始化

系统支持普通用户和管理员两种角色，管理员可审核纠错反馈。

### 方法一：直接修改数据库（推荐）

```bash
# 进入 Python 交互环境
python

# 执行以下代码
from app import create_app, db
from app.models import User
from werkzeug.security import generate_password_hash

app = create_app()
with app.app_context():
    # 查找第一个注册用户或指定用户
    user = User.query.first()
    if user:
        user.user_type = 'super'  # 设置为管理员
        db.session.commit()
        print(f"用户 {user.nickname} 已设为管理员")
    else:
        print("暂无用户，请先注册")
```

### 方法二：注册后手动修改

1. 正常注册一个用户
2. 使用数据库工具（如 DB Browser for SQLite）打开 `instance/app.db`
3. 找到 `users` 表，将对应用户的 `user_type` 字段从 `'normal'` 改为 `'super'`
4. 保存并重启服务

### 验证管理员权限

登录管理员账号后，访问：
- 审核页面：http://127.0.0.1:5000/admin/review
- 应能看到待审核的纠错反馈列表
